# 만성질환 맞춤 식이 분석 대시보드

식품의약품안전처 **식품영양성분DB**(가공식품)를 중심 데이터로, 질병관리청 **국민건강영양조사(KNHANES)**를 보조 데이터로 결합해 당뇨·고혈압·비만 등 만성질환 기준으로 음식을 태깅·검수·탐색하는 웹 대시보드입니다. 외부 라이브러리·서버 없이 동작하는 **단일 HTML 파일**로 구현되었습니다.

**🔗 라이브 데모: https://symin0729.github.io/Food-Nutrition_Dashboard_projects/**

---

## 주요 기능

데이터를 관리하는 **관리자(영양사·데이터 담당) 뷰**와 식이 정보를 조회하는 **사용자 뷰**로 분리됩니다.

### 관리자 뷰 (7개 탭)
- **개요** — 데이터 품질 KPI, 품질 구성 도넛, 질환별 태그 분포 막대
- **영양성분 검수** — 자동 플래그 큐(에너지 불일치·나트륨/당류 이상치·결측) 처리
- **표준화·중복** — 동일 정규명·복수 코드 클러스터의 대표 코드 지정
- **태그 기준 관리** — 질환별 임계 편집, 변경 영향 시뮬레이션, 버전 이력
- **KNHANES 인사이트** — 질환군·성연령·연도 추이 시각화
- **KNHANES 음식 검수 / 표준화**

### 사용자 뷰 (4개 탭)
- **식품영양성분 검색** — 가공식품 검색 + 질환별 적합/주의/제한 태그, 상세·대체식품 추천 (일반인/환자 모드 토글)
- **국민 평균 비교** — 동일 성·연령 집단(가중치 적용) 분포 대비 내 섭취 백분위
- **KNHANES 음식 검색**
- **식단 기록** — 누적 기록, 1일 기준 대비 초과 경고, CSV 내보내기

## 태그 판정 로직

판정 단위는 **표준 1회 제공량**(식약처 별지3 기준, 미분류 식품은 100g/100ml)이며 태그는 3단계(적합/주의/제한)입니다.

- **환자 모드** — 질환별 고정 제한 (예: 고혈압→나트륨 주의 300 / 제한 600mg)
- **일반인 모드** — 성·연령 프로필로 KDRI 2025 하루 기준(EER·CDRR 등)을 계산해, 1회 제공량이 하루 기준의 15% 이상=주의, 30% 이상=제한

## 데이터 출처

| 데이터 | 출처 |
|---|---|
| 전국통합식품영양성분정보 표준데이터 | 공공데이터포털 (15100064) |
| 국민건강영양조사 원시자료 (2020–2024) | 질병관리청 knhanes.kdca.go.kr |
| 식품별 1회 제공기준량 | 식약처 「식품산업체를 위한 영양표시 안내서」 |
| 2025 한국인 영양소 섭취기준(KDRI) | 보건복지부·한국영양학회 |

> 원시자료는 회원가입·약관동의가 필요해 저장소에 포함하지 않습니다. 정제 결과(`data/*.json`)와 대시보드에 임베드된 데이터만 공개됩니다.

## 실행 방법

별도 빌드·설치가 필요 없습니다. `index.html`(또는 `만성질환_식이분석_대시보드.html`)을 브라우저에서 열면 됩니다. 데이터가 파일에 임베드되어 있어 서버가 필요 없습니다.

## 기술 구성

- **단일 HTML** — 외부 스크립트/CDN 미사용
- **자체 SVG 차트** — 다계열 막대·라인·도넛을 직접 구현
- **프레임워크 없는 렌더링** — 전역 `DATA`(KNHANES) / `PRODUCTS`(식품DB), `render()`가 탭별 뷰 함수로 화면 교체
- 데이터 정제 파이프라인은 Python + `pyreadstat` 사용 (상세: [재현_프롬프트_전체.md](재현_프롬프트_전체.md))
